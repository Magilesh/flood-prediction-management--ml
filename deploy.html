🚀 Deploy Your Flood Prediction System

Time required: 5 minutes

Follow these steps to deploy your AI-powered flood prediction system to the cloud. Everything is pre-configured!

✅ Step 1: Deploy Backend to Railway (2 min)

Railway will host your FastAPI backend with the ML model.

🚂 Open Railway
  1. Click "Start a new project"
  2. Click "Continue with GitHub"
  3. Click "Deploy from GitHub repo"
  4. Search and select: Magilesh/flood-prediction-management--ml
  5. Click "Deploy now"
  6. Wait 2-3 minutes for deployment
  7. Copy your backend URL (e.g., https://flood-prediction.up.railway.app)

ℹ️ Step 2: Configure Email Notifications (1 min)

Optional: Enable automatic alerts when flood risk is high.

  1. In Railway dashboard, click your project
  2. Click "Variables" tab
  3. Click "Add Variable" for each:
    4. AUTHORITY_EMAIL=your-email@gmail.com
    SMTP_SERVER=smtp.gmail.com
    SMTP_PORT=587
    SMTP_USERNAME=your-email@gmail.com
    SMTP_PASSWORD=your-gmail-app-password
  4. Click "Deploy" to apply changes

For Gmail: Get app password at myaccount.google.com/apppasswords

✅ Step 3: Deploy Frontend to Vercel (2 min)

Vercel will host your Streamlit dashboard.

▲ Open Vercel
  1. Click "Continue with GitHub"
  2. Click "New Project"
  3. Select your repository
  4. Configure settings:
    5. Framework Preset: Other
    Root Directory: ./frontend
    Build Command: pip install -r ../requirements.txt && echo "Ready"
  5. Add Environment Variable:
    6. Name: BACKEND
    Value: [paste-your-railway-url-here]
  6. Click "Deploy"
  7. Wait 2-3 minutes for deployment

🎉 SUCCESS! Your Live Application

After deployment, your URLs will be:

🌐 Frontend Dashboard: https://flood-prediction.vercel.app
🔧 Backend API: https://[your-project].up.railway.app
📚 API Docs: https://[your-project].up.railway.app/docs

Test it: Open the frontend URL and click "Get Prediction"
